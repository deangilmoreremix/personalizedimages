VideoRemix Email Templates Guide

Overview

VideoRemix includes professionally designed, responsive HTML email templates for various authentication and notification purposes. These templates maintain the VideoRemix brand identity and are optimized for compatibility across email clients.

Available Templates

  • Confirm Signup
  • Invite User
  • Magic Link
  • Change Email Address
  • Reset Password
  • Reauthentication

Using the Templates

Template Locations

All email templates are located in the public/emails/ directory:

public/emails/
├── confirm-signup.html
├── invite-user.html
├── magic-link.html
├── change-email.html
├── reset-password.html
└── reauthentication.html

Personalization Tokens

The templates use double curly braces as placeholders for dynamic content. For example: {{firstName}}

Token Description Example
{{firstName}} User's first name Sarah
{{confirmUrl}} Confirmation URL https://videoremix.com/confirm?token=xyz
{{inviteUrl}} Invite acceptance URL https://videoremix.com/invite?token=xyz
{{magicLinkUrl}} Magic link authentication URL https://videoremix.com/auth/magic?token=xyz
{{resetUrl}} Password reset URL https://videoremix.com/reset-password?token=xyz
{{verifyUrl}} Verification URL for reauthentication https://videoremix.com/verify?token=xyz

Integration with Supabase Auth

Setting Up Custom Email Templates

To use these custom email templates with Supabase Auth:

  1. Navigate to your Supabase project dashboard
  2. Go to Authentication > Email Templates
  3. Select the template type you want to customize
  4. Paste the HTML content from the corresponding template file
  5. Update the message subject if needed
  6. Save the changes

Note: Supabase provides its own set of template variables that will replace our placeholder variables. You may need to adjust the variable names to match what Supabase expects, such as {{ .Email }} instead of {{email}}.

Customization

Modifying Templates

These templates are designed to be easily customizable:

  • Colors: Update the gradient colors in the header class and the button class
  • Logo: Replace the placeholder image with your actual logo
  • Content: Modify the text to match your voice and branding
  • Footer: Update links and copyright information

Warning: When modifying email templates, always maintain the table-based structure and inline styles to ensure compatibility across email clients. Avoid using CSS features that aren't widely supported in email clients.

Testing Email Templates

Best Practices for Email Testing

Before deploying your customized email templates:

  1. Test in multiple email clients (Gmail, Outlook, Apple Mail, etc.)
  2. Check both desktop and mobile rendering
  3. Verify all links work correctly
  4. Ensure personalization tokens are properly replaced
  5. Consider using email testing services like Litmus or Email on Acid

Accessibility Considerations

  • Maintain proper color contrast for readability
  • Use appropriate font sizes (minimum 14px for body text)
  • Include alternative text for all images
  • Ensure links have descriptive text
  • Test with screen readers if possible
